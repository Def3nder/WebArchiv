## SSH-Key Problem

### Symptom

WinSCP meldete:

```text
Die Gegenstelle hat unseren Schlüssel abgelehnt
```

### Ursache

Der Inhalt von:

```text
/root/.ssh/authorized_keys
```

war im SSH2/RFC4716-Format gespeichert:

```text
---- BEGIN SSH2 PUBLIC KEY ----
Comment: "eddsa-key-20260529"
...
---- END SSH2 PUBLIC KEY ----
```

OpenSSH akzeptiert dieses Format nicht.

### Korrektes Format

```text
ssh-ed25519 AAAA.... kommentar
```

in einer einzigen Zeile.

---

## Verzeichnisrechte

```bash
chmod 700 /root/.ssh
chmod 600 /root/.ssh/authorized_keys
```

# Datenablage

## Externe Disk

Eine externe Disk wurde in den Container eingebunden.

Mountpunkt:

```text
/opt/nodeapp/www
```

---

## Besonderheit

Das Dateisystem enthält:

```text
lost+found
```

Eigentümer:

```text
nobody:nobody
```

---

## Problem

Folgender Befehl:

```bash
chown -R www-data:www-data /opt/nodeapp
```

liefert:

```text
Permission denied
```

für:

```text
/opt/nodeapp/www/lost+found
```

---

## Wahrscheinliche Ursache

Die Disk wurde auf dem Proxmox-Host eingebunden und per Bind-Mount in den LXC durchgereicht.

Dadurch gelten die Host-Dateirechte.

Bei einem unprivilegierten LXC gilt:

```text
root im Container != root auf dem Host
```

Container-Root kann daher möglicherweise nicht alle Dateien ändern.

---

## Erkenntnis

Das Verzeichnis:

```text
lost+found
```

wird von ext4 automatisch angelegt.

Es wird von der Anwendung nicht benötigt.

Es sollte nicht verändert oder gelöscht werden.

---

## Empfohlene Struktur

```text
/opt/nodeapp/www
│
├── lost+found
└── data
```

Anwendungsdaten ausschließlich in:

```text
/opt/nodeapp/www/data
```
